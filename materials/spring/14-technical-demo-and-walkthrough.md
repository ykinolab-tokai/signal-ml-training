# 第14回 技術デモ・code walkthrough・知見整理

- 対象: B4・M
- 種別: 前期発展枠 / 固定

## この回の目標
- 自分の実装を、入力・処理・出力の流れに沿って 5 分程度で説明できる。
- 再実行に必要な情報と、他者に共有するには不足している情報を区別できる。
- 実装メモではなく、再利用可能な知見として何を残すべきかを判断できる。

## ミニ講義
- 技術デモでは「何を作ったか」よりも「どの入力に対して、どの処理を通し、何が出力されるか」を先に説明すると伝わりやすい。shape、保存物、実行コマンドはその骨格になる。
- code walkthrough では、全行を読む必要はない。処理の分岐点、前提条件、失敗しやすい箇所、再実行に必要な依存関係を押さえると、聞き手は自分で追試しやすくなる。
- 知見整理では、個人メモと共有資産を分けて考える。個人の一時的な試行錯誤と、他の学生が再利用できる説明や補助コードは価値が異なる。

## 演習
### 基礎レベル
1. 前期で自分が実装した script を 1 つ選び、`session14_demo_notes.md` に `## 選んだ script`, `## 入力`, `## 処理`, `## 出力`, `## 再実行コマンド` を書く。
2. 5 分で説明するつもりで、入力 shape、主要な処理、保存物 file 名、再実行に必要な前提を整理する。
3. `session14_demo_checklist.md` を作成し、説明に必要な項目を 6 個挙げ、それぞれ `OK` / `NG` を付ける。
4. `session14_reuse_candidates.md` を作成し、個人メモとして残すものと、他の学生が再利用できる形に整えるべきものを分けて書く。

### 発展レベル
1. 聞き手から出そうな質問を 3 つ作り、それぞれどの file や出力を見れば答えられるかを書く。
2. B4 または M は、選んだ script を来年度の教材として再利用するなら何を削り、何を補うべきかを 3 行以内で提案する。

## 確認ポイント
- `session14_demo_notes.md` に選んだ script、入力、処理、出力、再実行コマンドが具体的に書かれている。
- `session14_demo_checklist.md` の 6 項目すべてに `OK` / `NG` が付いている。
- 選んだ script の説明が、入力と出力の対応に触れている。
- `session14_reuse_candidates.md` の各項目に判断理由があり、単なる分類だけで終わっていない。
- B4 または M の発展課題で、優先基準や判断基準が自分の言葉で書かれている。
